# Security Policy

## Reporting a vulnerability

Do not open public issues for security-sensitive reports.

Report privately to maintainers with:

- affected component/path,
- reproduction steps,
- impact assessment,
- suggested mitigation (if any).

## Scope

Security review is required for:

- new integrations/adapters,
- scripts that read/write repository files,
- template packs distributed to downstream repos,
- changes that affect authentication, tenancy, secrets handling, or CI permissions.

## Disclosure policy

- We validate and triage reports privately.
- We coordinate fixes and only then disclose details.
- We avoid publishing exploit details before patch availability.
