# Create PRD

Use this starter when requirements need to become an implementable specification.

A useful PRD should include:

- context and strict scope;
- acceptance criteria with evidence;
- implementation slices with validation commands;
- risks, non-goals, and open questions;
- a clear definition of done.

Keep local domain rules in the repo overlay, not in portable core.
