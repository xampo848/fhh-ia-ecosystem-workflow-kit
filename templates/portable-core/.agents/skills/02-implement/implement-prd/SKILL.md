# Implement PRD

Use this starter when an approved PRD needs implementation.

## Core contract

1. Read the PRD completely.
2. Classify implementation size and risk.
3. Create or update a task tracker for non-trivial work.
4. Implement one small slice at a time.
5. Validate each slice before expanding scope.
6. Wait for subagents or delegated work to finish before dependent work continues.
7. Close only when acceptance criteria have evidence.

Project-specific test commands, architecture rules, and pattern skills belong in the repo overlay.
